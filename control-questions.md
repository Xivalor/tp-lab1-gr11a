# Контрольные вопросы

## 1. Какая текущая версия macOS?
На момент написания (февраль 2026) последней стабильной версией macOS является **macOS 13.3 Ventura**.

## 2. На основе какой UNIX-подобной системы построена операционная система macOS?
macOS основана на **BSD (Berkeley Software Distribution)**, которая является производной от Unix.

## 3. К какому типу относится ядро macOS? В чем отличия между XNU и Mach?
Ядро macOS относится к типу **монолитных ядер** с элементами **микроядра**.  
**XNU** (X is Not Unix) — это ядро macOS, которое сочетает в себе элементы Mach (микроядро) и компоненты BSD (монолитное ядро).  
Основное отличие между **XNU** и **Mach** заключается в том, что Mach является лишь основой для XNU, предоставляя поддержку многозадачности и виртуализации, а BSD предоставляет функции файловой системы, сети и другие сервисы.

## 4. Какой тип архитектуры у процессоров линейки M1 и выше, производимых Apple?
Процессоры Apple M1 и выше используют архитектуру **ARM** (в частности, **ARM64**).

## 5. Что такое виртуализация? Приведите примеры гипервизоров.
**Виртуализация** — это технология, которая позволяет запускать несколько операционных систем на одном физическом компьютере.  
Примеры гипервизоров:  
- **VMware**  
- **VirtualBox**  
- **Hyper-V**

## 6. Что такое «командная оболочка» / «командный интерпретатор»?
**Командная оболочка (или командный интерпретатор)** — это программа, которая позволяет пользователю взаимодействовать с операционной системой через текстовые команды. Она интерпретирует введенные команды и передает их системе для выполнения. Примером является **bash** или **zsh** в macOS.

## 7. Как называется командный интерпретатор и какая его версия в используемой операционной системе?
По умолчанию в macOS используется командный интерпретатор **zsh**. Версию можно проверить с помощью команды:
```bash
zsh --version
```

## 8. Для каких задач предназначено приложение Terminal в macOS?
Приложение **Terminal** в macOS используется для ввода команд, выполнения скриптов и взаимодействия с операционной системой через текстовый интерфейс. Это основной инструмент для работы с командной строкой.

## 9. Приведите полный путь к домашнему каталогу пользователя, под которым вы авторизованы в системе. Привести пример команды, позволяющей определить путь к домашнему (текущему) каталогу в консоли.
Путь к домашнему каталогу пользователя:
```bash
/Users/your_username
```
Для определения пути к домашнему каталогу в консоли используйте команду:
```bash
echo $HOME
```

## 10. Какой командой можем вывести содержимое каталога системных конфигурационных файлов?
Чтобы вывести содержимое каталога с системными конфигурационными файлами, используйте команду:
```bash
ls /etc
```

## 11. Приведите синтаксис команды для установки файлового менеджера Midnight Commander из консоли в macOS. Привести три варианта команды установки из консоли приложения Terminal в macOS.
Для установки **Midnight Commander** используйте одну из следующих команд:
1. С использованием **Homebrew**:
   ```bash
   brew install mc
   ```
2. С использованием **MacPorts**:
   ```bash
   sudo port install mc
   ```
3. С использованием **Fink**:
   ```bash
   fink install mc
   ```

## 12. Как можно вывести текст руководства (справочной информации) для какой-либо команды в консоли?
Для вывода справочной информации используйте команду `man`:
```bash
man команда
```

## 13. Для каких целей используется команда open?
Команда **open** в macOS используется для открытия файлов, директорий и приложений. Например, для открытия файла:
```bash
open myfile.txt
```

## 14. Приведите примеры команд группировки и перенаправления, используемых в командном интерпретаторе zsh.
Примеры:
- **Группировка команд**:
  ```bash
  (command1; command2)
  ```
- **Перенаправление вывода в файл**:
  ```bash
  echo "Hello" > output.txt
  ```

## 15. С помощью какой команды можем изменить командный интерпретатор с zsh на bash?
Для изменения командного интерпретатора с **zsh** на **bash** используйте команду:
```bash
chsh -s /bin/bash
```

## 16. В каких случаях выполняется инициализация репозитория? Какой командой?
Репозиторий инициализируется с помощью команды:
```bash
git init
```
Это создаст новый репозиторий в текущем каталоге.

## 17. Что такое коммит?
**Коммит** — это операция, которая фиксирует изменения в репозитории. Он сохраняет текущее состояние файлов и позволяет отслеживать изменения.

## 18. Почему перед созданием новой ветки важно создать коммит?
Создание коммита перед созданием новой ветки позволяет зафиксировать текущее состояние работы, обеспечивая чистую точку отсчета для переключения на другую ветку.

## 19. В каких состояниях может находиться файл в репозитории? Как происходит изменение состояния файла?
Файл может находиться в следующих состояниях:
- **Untracked** (неотслеживаемый)
- **Unmodified** (неизмененный)
- **Modified** (измененный)
- **Staged** (подготовленный к коммиту)

Для изменения состояния файла используются команды `git add` и `git commit`.

## 20. Что такое ветка?
**Ветка** — это независимая линия разработки в репозитории, которая позволяет работать над новыми функциями или исправлениями, не влияя на основную ветку.

## 21. Какое условие является обязательным для формирования ветки master после клонирования репозитория или его инициализации?
Для формирования ветки **master** нужно выполнить хотя бы один **коммит**.

## 22. Что такое HEAD?
**HEAD** — это указатель на текущую ветку или последний коммит в репозитории. Он всегда указывает на состояние рабочей директории.

## 23. Способы создания веток.
Ветки можно создать с помощью команды:
```bash
git branch имя_ветки
```
или с переключением на неё:
```bash
git checkout -b имя_ветки
```

## 24. Как узнать текущую ветку?
Для определения текущей ветки используйте команду:
```bash
git branch
```

## 25. Как переключаться между ветками?
Для переключения между ветками используйте команду:
```bash
git checkout имя_ветки
```

## 26. С помощью какой команды можно слить две ветки?
Для слияния веток используйте команду:
```bash
git merge имя_ветки
```

## 27. Как подключить к инициализированному (созданному) локальному репозиторию внешний репозиторий? Какой командой?
Для подключения внешнего репозитория используйте команду:
```bash
git remote add origin url_репозитория
```

## 28. Какая команда позволяет отменить последний коммит?
Чтобы отменить последний коммит, используйте команду:
```bash
git reset --soft HEAD~1
```

## 29. Для чего предназначена команда rebase?
Команда **rebase** используется для изменения базы ветки. Она позволяет переместить или перенести изменения с одной ветки на другую.

## 30. В каких случаях требуется создание форка? Приведите способы создания форка, включая консольную команду.
**Форк** создается, когда нужно внести изменения в чужой репозиторий, не имея прямого доступа к его исходному коду. Форк можно создать на GitHub, нажав **Fork**, либо через консоль:
```bash
git clone https://github.com/original_owner/repository.git
```
